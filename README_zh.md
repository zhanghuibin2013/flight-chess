# 导弹飞行棋

一款多人在线策略棋盘游戏，融合空战机制，包含导弹系统、雷达系统和战术玩法。基于 Node.js、Socket.IO、React 和 TypeScript 构建。

## 🎮 游戏概述

《导弹飞行棋》是经典飞行棋棋盘游戏的数字化改编版本，融合了现代战斗元素，包括：

- **空空导弹 (AAM)**：与敌方战机进行空中格斗
- **防空导弹 (SAM)**：通过雷达引导拦截进入防空区的敌机
- **反辐射导弹 (ARM)**：摧毁敌方雷达系统
- **巡航导弹**：远程精确打击脆弱目标
- **雷达系统**：扩展探测区域，增强防御能力
- **知识问答挑战**：测试知识获取奖励或惩罚

## ✨ 功能特性

- **实时多人对战**：2-4 名玩家通过 WebSocket 实时竞技
- **权威服务器**：所有游戏逻辑在服务器端验证，防止作弊
- **策略性战斗**：多种武器系统，各具独特机制
- **动态 gameplay**：随机事件、知识问答挑战和奖惩卡牌
- **响应式设计**：基于 React 和现代 CSS 构建的美观界面
- **跨平台**：在任何带有网页浏览器的设备上游玩

## 🚀 快速开始

### 前置要求

- Node.js ≥ 18
- npm ≥ 9

### 安装

```bash
# 克隆仓库
git clone <repository-url>
cd missile-flying-chess

# 安装依赖
npm install

# 构建所有包
npm run build
```

### 开发模式

在 separate 终端中运行以下命令：

```bash
# 终端 1：启动游戏服务器（端口 3001）
npm -w @fkzz/server run dev

# 终端 2：启动 Web 客户端（端口 5173）
npm -w @fkzz/web run dev
```

在多个浏览器标签页/窗口中打开 http://localhost:5173 与朋友一起游玩。

### 生产模式

```bash
# 构建项目
npm run build

# 启动生产服务器（在端口 3001 上同时提供 API 和静态文件服务）
npm -w @fkzz/server start
```

访问 http://localhost:3001 开始游戏。

## 🎯 游戏玩法

### 1. 大厅设置
- 在大厅屏幕输入昵称
- 一名玩家创建房间（生成 6 位字符代码）
- 其他玩家使用房间代码加入
- 每位玩家选择颜色座位（红/黄/蓝/绿）并点击**准备**

### 2. 游戏开始
- 当所有玩家准备就绪后（至少 2 名玩家），房主点击**开始游戏**
- 玩家轮流掷骰子并移动战机

### 3. 核心机制

#### 起飞
- 掷出所需数字（可配置：困难模式为 6，普通模式为 5-6，简单模式为 2-4-6）
- 将战机从机库移动到起飞点
- 掷出 6 可获得额外回合

#### 移动
- 战机根据骰子点数在棋盘上顺时针移动
- 停留在特殊功能格子上触发事件（导弹工厂、雷达工厂、图书馆）

#### 跳跃机制
- 降落在同色格子上时自动跳跃到下一个匹配格子
- 每次移动序列只能跳跃一次

#### 快速通道
- 主动进入：飞过通道并触发跳跃
- 被动进入（通过跳跃）：飞过通道但不触发额外跳跃

#### 降落区
- 终点前 4 格为受保护区域（免疫大多数攻击）
- 必须掷出精确数字才能降落；超出会导致后退

#### 编队（迭机）
- 多架同色战机停留在同一格子形成编队
- 遭遇编队时适用特殊碰撞规则

#### 碰撞
- 敌方战机在同一格子：双方都返回机库
- 编队碰撞：攻击者和一名防御者返回机库

### 4. 武器系统

#### 雷达卡
- 决定防空识别区大小
- 0 张卡：无法使用防空导弹
- 1 张卡：1 格区域
- 3 张卡：3 格扇形区（半径 2）
- 5 张卡：5 格扇形区（半径 3）
- 7 张卡：7 格扇形区（半径 3）
- 偶数不扩大区域

#### 空空导弹 (AAM)
- 前方 4 格内出现敌机时触发
- 双方掷骰子，点数大者获胜
- 胜者将败者送回机库
- 如果防御方拥有 AAM 卡，可以反击

#### 防空导弹 (SAM)
- 敌机进入雷达区时自动提示
- 需要至少 1 张雷达卡才能使用

#### 反辐射导弹 (ARM)
- 攻击敌方雷达系统
- 掷出 5 或 6 成功命中
- 摧毁一张敌方雷达卡

#### 巡航导弹
- 攻击起飞点或降落区的战机
- 起飞点目标：自动命中
- 降落区目标：掷出 4-6 成功

### 5. 装备获取

#### 导弹工厂
- 降落在"导弹工厂"格子抽取随机导弹卡
- 碰撞状态下无法抽取

#### 雷达工厂
- 降落在"雷达工厂"格子获得 1 张雷达卡
- 碰撞状态下无法抽取

### 6. 知识问答系统

#### 图书馆挑战
- 降落在"图书馆"格子触发问答
- 回答正确：抽取奖励卡
- 回答错误：抽取惩罚卡

#### 奖励卡（共 30 张）
- 额外掷骰并前进（×4）
- 前进 2/4/6 格（各 ×4）
- 获得导弹/雷达（各 ×4）
- 跳过对手回合（×4）
- 防御一次攻击（×2）

#### 惩罚卡（共 30 张）
- 额外掷骰并后退（×4）
- 后退 2/4/6 格（各 ×4）
- 返回起飞点（×2）
- 停止一轮（×4）
- 损失导弹/雷达（各 ×4）

### 7. 胜利条件

**模式一：竞速胜利**
- 率先有 2 架战机抵达终点的玩家获胜

**模式二：限时胜利**
- 设定时间限制（建议 2 小时）
- 规定时间内抵达终点的战机数量最多的玩家获胜

## 📝 知识问答题库

游戏包含知识问答系统。通过编辑 `data/questions.json` 自定义问题：

```json
[
  {
    "id": "q-001",
    "prompt": "AAM 代表什么？",
    "options": [
      "空空导弹",
      "防空导弹",
      "自动瞄准模块",
      "航天航空标记"
    ],
    "answerIndex": 0
  }
]
```

**字段说明：**
- `id`：唯一标识符
- `prompt`：问题文本
- `options`：答案选项数组（通常 4 个）
- `answerIndex`：正确答案的从零开始索引

修改文件后重启服务器。

## 🛠️ 技术栈

| 层级 | 技术 |
|------|------|
| **协议** | TypeScript + Zod  schema 验证 |
| **服务器** | Node.js 18, Socket.IO, nanoid, ESM 模块 |
| **游戏引擎** | 权威状态机，使用 structuredClone 快照 |
| **客户端** | React 18, Vite 5, Zustand 状态管理, Socket.IO 客户端 |
| **随机数** | 服务器端 `crypto.randomInt`（无客户端随机） |
| **样式** | 响应式设计的纯 CSS |

## 🔒 安全与设计

- **权威服务器**：客户端仅请求操作；服务器根据游戏状态验证所有移动
- **私有抽卡**：只有抽卡玩家看到卡片详情；其他人看到通用日志
- **无客户端 RNG**：所有随机数在服务器端生成，防止操纵
- **状态快照**：每次操作后将完整游戏状态同步给所有客户端
